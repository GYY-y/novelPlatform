# Novel Writing: 中文长篇创作助手

## 流程总览

1) 需求收集：用 AskUserQuestion 逐一询问题材/主角设定/性格/核心冲突/章节数/基调与受众。  
2) 规划确认：生成大纲与人物表，向用户展示摘要并获得确认。  
3) 逐章创作：按模板写作，达成字数与钩子要求，完成后更新摘要与状态。  
4) 复盘润色：检查连贯性、节奏和口语化程度，必要时扩写。  

## 需求收集（每问提供选项）

- 题材：悬疑推理 / 科幻未来 / 奇幻玄幻 / 现代言情 / 古代言情 / 武侠仙侠 / 历史架空 / 都市现实  
- 主角设定：男 / 女 / 双主角 / 群像；如需继续问职业/身份与年龄。  
- 主角性格：热血正义 / 冷静智谋 / 温暖治愈 / 高冷孤傲 / 阴暗腹黑 / 成长逆袭  
- 核心冲突：生死存亡 / 查明真相 / 爱情阻碍 / 复仇雪恨 / 权力争夺 / 成长突破 / 守护保护  
- 章节数量：10 / 15 / 20 / 30 / 50 / 自定义  
- 基调与受众：明快成长 / 黑暗悬疑 / 轻松日常 / 成人向深度 / 青少年向  

收集完毕后总结需求并得到用户确认，再进入规划。

## 规划阶段

1) 创建项目文件夹：`novels/<作品名>/`  
2) 生成 `00-大纲.md`：章节列表、每章300-500字摘要、TODO 列表标记状态。  
3) 生成 `01-人物档案.md`：主角、反派、配角的背景/目标/弱点/弧光。  
4) 将规划摘要展示给用户并获得确认，确认后进入创作。

## 逐章创作流程

1) 读大纲：查看 TODO 与上一章摘要，标记本章为进行中。  
2) 设计开头钩子：前20%必须出现冲突或悬念。  
3) 场景规划：为本章列出3-5个关键场景。  
4) 写作：使用章节模板，正文控制在3000-5000字；确保对话口语化、动作驱动，避免空洞形容。  
5) 结尾钩子：埋悬念或反转，引向下一章。  
6) 字数检查：使用脚本 `python scripts/check_chapter_wordcount.py <章节路径>`，不足3000字则扩写。  
7) 连贯性与润色：检查人物动机、节奏与细节，删冗余、增具体描写，减少四字成语堆砌。  
8) 更新大纲：在 `00-大纲.md` 添加本章摘要（300-500字），将状态设为完成。

## 写作要点

- 展示而非讲述：用动作/对话驱动，不用笼统情绪描述。  
- 冲突贯穿：每章至少一个显性冲突或决策点。  
- 视角稳定：保持叙事视角一致，必要时明确转换。  
- 节奏切换：长短句交替，场景间留出呼吸点与转场。  
- 对话个性：不同角色有独特语气、词汇与态度。  
- 钩子承上启下：开头有即时吸引力，结尾留疑问或风险。
